# Phase 1 Prompt：术语扫描

## 目标

从章节文本中识别"核心概念"——理解本书时不可或缺、作者反复使用或特意定义的**术语、人名、引用作品**。

这一阶段产出 concepts 列表，作为 Phase 2（翻译 + 出现标注）的术语表基础。

---

## Prompt 全文

````
你是一个精确的学术术语抽取助手, 服务于一个"书的索引库"(Book Language Server).
你的任务是: 从给定章节文本中识别"核心概念".

## 什么是"核心概念"

必须抽取:
1. 作者特意定义、反复使用、有专门含义的术语 (包括作者自创的)
2. 被作者引用其观点/研究/理论的人名 (研究者、学者、历史人物)
3. 被引用的作品: 书籍、论文、理论流派、具名研究

不要抽取:
1. 常识词 / 过于泛泛的词 (如"家长"、"孩子"、"爱"、"问题")
2. 例子里偶然出现的人名 (如故事中的某个孩子)
3. 普通描述中出现的物品/动作
4. 作者本人(除非被自我引用)

## 分类体系

- term: 抽象术语/概念
- term_custom: 作者自创或特殊用法的术语
- person: 人名
- work: 书籍/论文/研究
- theory: 理论流派/学派

## 输出要求

严格 JSON, 不要任何解释性前后文:

{
  "concepts": [
    {
      "term": "规范名 (作者最常用的写法)",
      "aliases": ["别名1", "别名2"],
      "category": "term" | "term_custom" | "person" | "work" | "theory",
      "initial_definition": "一句话定义, 尽量用作者的话",
      "first_occurrence_paragraph": "P01" 格式,
      "confidence": 0.0-1.0,
      "reasoning": "一句话说明为什么抽取"
    }
  ]
}

## 数量原则

- 宁少勿多. 每 10 段文本 ≤ 5 个概念是个好比例.
- confidence 反映"这是本章核心概念"的把握度, 不是"它存在"的把握度.
- 对不确定是否足够核心的, 用低 confidence (< 0.6) 标注, 不要漏掉也不要硬撑.

## 章节文本

{章节文本, 每段用 [P##|pid] 前缀}

---

现在开始抽取. 直接输出 JSON, 不要任何前后文.
````

## 设计考量

### 为什么要 `reasoning` 字段

帮助 debug：如果 LLM 抽出了奇怪的概念，看 reasoning 能立刻知道它想什么。上线前可以去掉以省 token。

### 为什么要 `confidence` 而不是二元 "抽/不抽"

- 模棱两可时不浪费信息（标低分让用户决定）
- 下游能按 confidence 排序/过滤
- 同一本书不同 run 可以比较一致性

### 为什么不在 Phase 1 做 occurrence 分类

Phase 1 只产出 concept 候选。Phase 2 才扫全文做：
- 每次出现的位置
- 类型标注（definition/refinement/usage）
- 翻译 + term_spans

分离的好处：
- Phase 1 可以用 Haiku（便宜），只需"读懂大意"
- Phase 2 需要 Sonnet（精度），但已有术语表作输入
- Phase 1 可以只扫摘要（省 tokens），Phase 2 扫全文

### 边界情况

- "阿比盖尔"（作者女儿，例子里的主角）：是人名但不是"被引用观点的学者"，应**不抽取**
- "行为主义"：是理论流派，category=theory
- "爱"：日常词不抽；但"父母爱/无条件的爱"属于作者的专门用法，可能 term_custom
- "撤回爱"：作者引用的某种养育手段，term
